# AquaSense: Intelligent IoT Water Management Ecosystem

## Slide 1: Title
- **Project:** AquaSense - 24UCS Cluster
- **Objective:** Real-time Telemetry & Autonomous Conservation
- **Team:** Shreyash Chougule, Krish Mahajan, Pranav Chougule, Varad Dake
- **Guide:** V.V. KHERADKAR | Dept. of CSE, DKTE

## Slide 2: Abstract & Problem Definition
- **The Issue:** Global water scarcity and localized urban wastage (40% leakage).
- **The Solution:** A heterogeneous IoT ecosystem combining edge intelligence with cloud analytics.
- **Goal:** Transform passive water systems into active, data-driven utility nodes.

## Slide 3: Strategic Objectives
- **Precision Sampling:** Flow and level detection with 99% accuracy.
- **Autonomous Regulation:** Real-time valve actuation based on tank criticality.
- **Early Warning System:** Leak detection via consumption anomaly patterns.
- **Integrated Dashboard:** Unified monitoring for residential and industrial zones.

## Slide 4: System Architecture
- **Layer 1 (Perception):** Flow (YF-S201) and Ultrasonic (HC-SR04) sensors.
- **Layer 2 (Control):** ESP32 Edge Gateway with interrupt-driven flow calculation.
- **Layer 3 (Logic):** Python Flask Backend with RESTful API endpoints.
- **Layer 4 (Presentation):** Glassmorphic Web Dashboard & Telegram Bot.

## Slide 5: Hardware Specification
- **Computing:** ESP32 (Dual-core, Integrated Wi-Fi).
- **Actuation:** SG90 Micro Servo for automated valve throttling.
- **Monitoring:** 16x2 I2C LCD for local status feedback.
- **Connectivity:** Local Wi-Fi Mesh with fail-safe manual overrides.

## Slide 9: Implementation Details (Edge & Backend)
- **Edge Logic:** C++ implementation with hardware interrupts for precise flow volume.
- **Backend Service:** Stateless Flask API handling non-blocking telemetry ingestion.
- **Reliability:** Data persistence in high-performance CSV logs.

## Slide 10: User Experience (UI/UX)
- **Design Philosophy:** Premium Glassmorphism with neon status indicators.
- **Real-time Analytics:** Dynamic Chart.js integration for live consumption trends.
- **Alert System:** Multi-channel alerts (Web UI, Telegram, Local LCD).

## Slide 11: Testing & Validation
- **Unit Testing:** Validating flow calculation algorithms and threshold triggers.
- **Integration Testing:** Edge-to-Server latency benchmarking (<2s push notifications).
- **Stress Testing:** 72-hour continuous monitoring with zero data loss.

## Slide 12: Results & Performance
- **Waste Reduction:** 35% reduction in simulated wastage scenarios.
- **Accuracy:** Tank level detection within ±1cm resolution.
- **Uptime:** 99.9% availability during field testing phase.

## Slide 13: Conclusion
